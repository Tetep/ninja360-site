Publish Checklist — ninja360.net

Every page, its slug, SEO title/meta, and the 301 from the old site. Run it top to bottom.

The sequence (do not reorder)

  1. Commit & push the repo so everything is versioned: git add -A && git commit -m "site build" && git push
  2. Publish each page in GHL from the table below. For every funnel step: paste the page HTML into a Custom JS/HTML element, set the SEO Title + Meta, set the slug, and uncheck Header and Footer in the step settings (our nav/footer is in the page body).
  3. Verify each live URL loads, is styled, nav works, and the audit buttons reach the form.
  4. Point the 301s (last column) in Cloudflare only after the new URLs are confirmed live.
  5. Submit the GHL sitemap in Google Search Console + use Change of Address so rankings move from ninja-360.com.
Confirm this before publishing the /work pages

The table uses nested paths like /work/restaurants. If your GHL funnel won't accept a slash in the step path, use the flat form work-restaurants instead — and update that row's 301 target to match the real published URL. Whatever GHL actually serves is what the 301 must point to.
